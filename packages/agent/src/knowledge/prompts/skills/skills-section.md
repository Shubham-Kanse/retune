# Skills Section Writing

## Critical Context
Only list skills the candidate genuinely has and could discuss or demonstrate in an interview. Do not add skills for keyword stuffing that the candidate would fail to back up under scrutiny. The skills section is the second thing recruiters read — and the first thing ATS systems index.

---

## Format

4–5 categories, comma-separated values within each. Bold category label followed by a tab, then the values.

```
**Business Analysis:**    Requirements gathering, stakeholder management, process mapping (BPMN), gap analysis
**Data & Reporting:**     SQL, Power BI, Excel (advanced), Tableau, dashboard design
**Methodologies:**        Agile, Scrum, Waterfall, SDLC, UAT coordination, sprint planning
**Tools:**                Jira, Confluence, Salesforce, ServiceNow, Visio, MS Office Suite
**Domain Knowledge:**     [Role-specific — e.g., financial systems, regulatory compliance, supply chain]
```

## Rules

1. **Use exact JD terminology** — not synonyms. "Power BI" not "PowerBI." "Requirements gathering" not "requirement elicitation" unless the JD uses that phrase.
2. **Mirror JD category names** when possible — if the JD says "Technical Skills" instead of "Data & Reporting," use their label.
3. **Order skills within each category by JD relevance** — the most critical keyword first in each line (ATS scores earlier items more heavily).
4. **Order categories by relevance to the role** — the most critical category appears first.
5. **Acronyms:** spell out on first use — "Business Process Model and Notation (BPMN)." Subsequent mentions: acronym only.
6. **Do not exceed 6–7 items per category** — long lists dilute keyword density and look padded.
7. **Primary JD keywords must appear here AND in at least one experience bullet** — orphan keywords in skills but nowhere else score poorly.

## Quality Checklist

- [ ] Every primary JD keyword appears in the skills section
- [ ] Skills are ordered by JD relevance (most important first)
- [ ] No skill listed that the candidate can't defend in an interview
- [ ] Exact JD terminology used (not synonyms)
- [ ] Acronyms expanded on first use
- [ ] No more than 5–6 items per category
- [ ] Formatting: bold category + tab + comma-separated values
- [ ] 4–5 categories total (not 2, not 8)
