# Resume Architecture Guide

## Critical Context
The candidate's reference resume(s) are **their own work** — the ground truth for their experience, voice, and history. Never fabricate, inflate, or attribute skills or achievements not present in the source material. Always read the candidate's background before writing anything. When specific numbers are unavailable, use verified ranges, scopes, or comparative metrics.

---

## Document Structure (Skills-First Format — 2026 Standard)

```
1. HEADER
2. PROFESSIONAL SUMMARY
3. SKILLS
4. WORK EXPERIENCE
5. EDUCATION
6. CERTIFICATIONS (if relevant to the role)
```

The Skills-First format (summary → skills → experience) outperforms the legacy Experience-First format in ATS systems and recruiter scan patterns. Skills appear on page 1 within 5 seconds of viewing.

---

## Section 1: Header

```
[CANDIDATE FULL NAME]
[Target Role Title]
[City, Country] | [email] | [phone] | LinkedIn
```

**Formatting rules:**
- Name: 16pt bold, centred, uppercase, navy blue (#1F3A5F)
- Designation (role title): 11pt, centred, on its own line below the name
- Contact: 9.5pt, centred — "LinkedIn" as a clickable hyperlink, not raw URL
- Location: use the **company's city** unless the role is remote — signals local availability
- Phone: format with country code for international roles (e.g., +353 89 977 5080 or +44 7911 123456)
- No photos, no icons, no graphics — ATS strips them and may corrupt surrounding text
- Contact info in the document **body**, never in Word headers/footers (invisible to ~40% of ATS)
- Right-to-work status: include only if the JD asks, or if the candidate is on a visa/permit that's relevant

---

## Section 2: Professional Summary

**Length:** 80–120 words exactly. Count them. Verify before finalising.

**Formula:**
```
Line 1: [Seniority] [Role] with [X years] experience in [industry/function]
Line 2: [Specific quantified achievement matching what THIS company cares about most]
Line 3: Expert in [top 4–5 skills pulled verbatim from the JD]
Line 4: [Unique differentiator — domain expertise, notable employer, or credential]
Line 5 (optional): [Cultural fit signal echoing the company's stated values]
```

**Calibration by seniority:**

| Level | Years | Line 1 anchor | Differentiator (Line 4) |
|---|---|---|---|
| Junior / Graduate | 0–2 | Recent graduate / early-career | Degree, dissertation, internship result |
| Mid-level | 2–6 | [N]+ years in [function] | Key domain, notable employer, measurable delivery |
| Senior | 6–12 | Senior [role] with [N] years | Leadership, scale, complex stakeholder landscape |
| Lead / Principal | 12+ | [Role] leader with [N] years | Org-level impact, team built, P&L ownership |

**Banned phrases (instant AI-detection flag):**
- "Passionate professional with a proven track record..."
- "Results-driven," "hardworking," "team player" without proof
- "Dynamic," "motivated," "enthusiastic," "exceptional," "outstanding"
- "Proven track record" in any form
- Any sentence that could be copy-pasted to a different candidate unchanged

## Section 3: Skills

Defer to the `skills-section` skill for full detail. Core rules:
- 4–5 categories, comma-separated within each
- Use **exact JD terminology** — not synonyms or paraphrases
- Order skills within each category by JD relevance (highest-weight keyword first)
- Only list skills the candidate genuinely has and can defend in an interview
- Skills-First placement means this section is read immediately after the summary

---

## Section 4: Work Experience

**Role header format:**
```
### JOB TITLE
#### Company Name | Month YYYY - Month YYYY
```

For current roles: `Month YYYY - Present`

**Date rules:**
- ASCII hyphens (-) for date ranges — never en-dashes (–) or em-dashes (—)
- Month YYYY format consistently throughout (e.g., "December 2021 - July 2024")
- Company/institution lines are rendered in italic (not bold) via the `####` markdown prefix

**Bullet construction:** Defer to the `bullet-writing` skill for full detail. Core rules:
- 4–6 bullets per role
- Every bullet has a quantified result or measurable scope
- Max 2 lines per bullet
- Never repeat the same verb twice in one role
- Never start with: "Responsible for," "Helped," "Assisted," "Worked on"
- "Supported" only when ownership context follows immediately

**Role count guidance:**
- Most recent role: 5–6 bullets (most relevant, most real estate)
- Second-most-recent role: 4–5 bullets
- Earlier roles: 3–4 bullets unless highly relevant to the JD
- Roles older than 10 years: 2–3 bullets max, or omit if irrelevant

---

## Section 5: Education

```
### DEGREE TITLE
#### University Name, City | Start Year - End Year
```

- Include GPA / grade classification only if strong (First Class, 3.8+, Dean's List) or explicitly requested
- Add relevant coursework only if it directly matches the JD and graduation was within 3 years
- For candidates 5+ years into their career, education appears after all work experience without elaboration

---

## Section 6: Certifications (if applicable)

```
### CERTIFICATION NAME — Issuing Body | Year
```

Include only active certifications relevant to the target role. Expired certifications omit unless completion date is recent and the credential is still recognised.

---

## Handling Skills from the JD

1. **If a skill is in the JD, include it.** Find the best way to frame it using the candidate's real experience, projects, or coursework. The candidate is applying to this role because they believe they can do it.
2. **If uncertain how to frame a skill**, ask the candidate directly — one specific question, then proceed.
3. **Never silently omit a JD skill.** Either include it or flag it to the candidate.
4. **Tier 1 skills** (required, explicitly stated) → full bullets with metrics
5. **Tier 2 skills** (preferred, mentioned once) → supporting mention in a bullet or skills line
6. **Tier 3 skills** (implied or culture-adjacent) → listed in Skills only

## Action Verb Library

Pick based on achievement type. Never repeat the same verb twice in one role. Mix categories across bullets to show breadth.

| Category | Verbs |
|---|---|
| Analysis | Analysed, Assessed, Evaluated, Investigated, Identified, Diagnosed, Mapped, Modelled, Forecasted, Interpreted, Examined, Uncovered |
| Delivery | Delivered, Implemented, Executed, Launched, Deployed, Produced, Completed, Coordinated, Facilitated |
| Improvement | Streamlined, Optimised, Reduced, Improved, Accelerated, Simplified, Transformed, Redesigned, Eliminated, Consolidated |
| Leadership | Led, Spearheaded, Championed, Directed, Guided, Mentored, Influenced, Negotiated, Presented |
| Creation | Designed, Developed, Created, Established, Formulated, Drafted, Authored, Devised, Structured |
| Stakeholder | Liaised, Collaborated, Partnered, Translated, Advised, Recommended, Aligned, Bridged |
| Technical | Engineered, Automated, Configured, Integrated, Migrated, Deployed, Monitored, Tested, Validated |
| Commercial | Grew, Increased, Expanded, Generated, Secured, Closed, Negotiated, Retained |

## Layout Validation

Before finalising:
- [ ] Page count: 1 page for <3 years experience; 1.5–2 pages for 3–10 years; 2 pages max for 10+ years
- [ ] First work experience bullet visible on page 1
- [ ] Section order: Header → Summary → Skills → Experience → Education → Certs
- [ ] No section is empty
- [ ] Consistent verb tense: past tense for past roles, present for current role
- [ ] Candidate's name appears only in the header (not scattered in bullets)
