# Charter 21 Epic 01 — Canonical Set Expansion

**Charter:** 21 — Eval Leadership
**Status:** Not started
**Owner:** Eval lead

## Goal

Grow `tests/cases.jsonl` from 14 → 200 cases covering the full breadth
of roles Retune supports, with hand-graded rubric reference for each
case. Without this, every other Charter 21 epic (regression gate, live
nightly, rubric calibration) operates on too-narrow a sample.

## Definition of Done

- `packages/eval/cases.jsonl` contains ≥ 200 cases.
- Each case has: `(resume_id, jd_text, expected_role_family, expected_seniority, expected_outcome, rubric_grades)`.
- Cases span 9 role families: software eng, product, design, data, marketing, ops, finance, healthcare, education.
- Cases span 3 seniority levels: junior, mid, senior.
- 25% of cases include explicit refusal expectations (insufficient evidence, role mismatch, fabricated claim).
- Per-case rubric reference is hand-graded by 2 reviewers with ≥ 0.8 inter-rater agreement.
- Source attribution: 50% from anonymised production runs, 50% from synthetic adversarial cases generated by Charter 26 Epic 02.

## Stories

### Story 1.1 — Source extraction
Pull 100 anonymised cases from production `generations` table where
`outcome != null`. Strip PII. Convert to `cases.jsonl` shape.

**Acceptance:** 100 cases land in a `production-extract.jsonl` review
file. PII scan (`stripPII` from `lib/onboarding-v2/llm/guardrails.ts`)
returns zero hits.

### Story 1.2 — Synthetic case generation
Generate 100 cases using Claude/GPT with prompts that vary role,
seniority, evidence quality, and adversarial framing. Each generated
case includes the prompt that produced it (for audit).

**Acceptance:** 100 cases in `synthetic-cases.jsonl`. Coverage report
shows even distribution across role families.

### Story 1.3 — Rubric grading session
2 reviewers grade all 200 cases against the rubric in
`packages/eval/src/rubrics/`. Disagreements resolved in a 1-hour
calibration meeting.

**Acceptance:** Inter-rater κ ≥ 0.8. All 200 cases have a final rubric grade.

### Story 1.4 — Merge + freeze
Merge the 200 cases into `packages/eval/cases.jsonl`. Tag the file
version: `cases-v2.0.0`. Future PRs that mutate it require a
quality-review check.

**Acceptance:** `git log packages/eval/cases.jsonl` shows the freeze
commit; `pnpm --filter @retune/eval eval --baseline-only` passes against
the new set.
